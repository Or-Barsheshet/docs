# Why Tagging Data is Useful?

Tags help you **quickly find and organize data items that are relevant to your task**.  

This can help you:

1. **Data Curation:** Select a subset of relevant data for a specific task, such as training a model. Label it as **“representative”** or classify it into groups like “train,” “test,” or “validation.”  
2. **Data Cleaning:** Identify and remove unwanted data by tagging it as **low quality, duplicates, or outliers**.  
3. **Workflow Management:** Highlight priority items using tags like **urgent** or **critical**. Assign review states such as **for review**, **approved**, or **rejected** when reviewing mislabeled items.  
4. **Experiment Tracking:** Tag datasets used in experiments (e.g., "Experiment-123") to track inputs and outputs.

***

# Managing Tags of Multiple Items

1. Open a dataset and explore its contents.  
2. Select the **Clusters**, **Images**, and **Objects** you wish to tag.  
3. The selected items are added to the "**Selected Items**" menu at the top right.  
4. Open the **"Selected Items"** menu and click on the **"Manage Tags"** button to associate an existing **User Tag** with the displayed items.  
5. Assign one or more **User Tags** to the selected items.

## Listing & Removing Tags

1. Any tags assigned to the Selected Items can be viewed in the Assigned Tags Menu.  
2. Selecting a tag allows you to **remove** it from any items it's attributed to — even if the tag isn’t applied to all items in a selected cluster, for example.
